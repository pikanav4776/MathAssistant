# MathAssistant v1.0 ‚ÄĒ Release Checklist

Use this checklist before tagging `v1.0.0` on production.

**Last updated:** June 22, 2026

---

## 1. Environment variables (Render)

Set on **both** `mathassistant-api` (production) and `mathassistant-api-staging`:

| Variable | Required | Notes |
|----------|----------|-------|
| `DATABASE_URL` | Yes | Neon connection string with `?sslmode=require` |
| `JWT_SECRET` | Yes | Strong random string; app **refuses to start** on Render if missing or dev default |
| `CORS_ORIGINS` | Yes | Exact frontend URL(s), comma-separated, no trailing slash |
| `SENTRY_DSN` | Optional | Error monitoring; omit to disable |
| `SENTRY_TRACES_SAMPLE_RATE` | Optional | Default `0.1` when Sentry enabled |
| `RATE_LIMIT_ENABLED` | Optional | Default `true` |

Frontend (`mathassistant-frontend`): `API_BASE_URL` ‚Üí backend HTTPS URL (build-time `VITE_API_BASE_URL`).

---

## 2. Staging smoke test

After deploy to staging:

- [ ] `GET /health` ‚Üí `200 {"status":"ok"}`
- [ ] `GET /ready` ‚Üí `200` with `"db":"connected"`
- [ ] Guest: start session ‚Üí submit step ‚Üí complete or give up
- [ ] Calculator: build expression ‚Üí Use Expression ‚Üí start session
- [ ] Auth: register ‚Üí login ‚Üí start session (owned) ‚Üí refresh page (resume)
- [ ] Starter problems panel: filter + click to start
- [ ] Unsupported problem shows friendly error
- [ ] Browser console: no CORS errors
