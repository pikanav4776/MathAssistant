# MathAssistant v1.0 — Release Checklist

Use this checklist before tagging `v1.0.0` on production.

**Last updated:** June 22, 2026

---

## 1. Environment variables (Render)

Set on **both** `mathassistant-api` (production) and `mathassistant-api-staging`:

| Variable | Required | Notes |
|----------|----------|-------|
| `DATABASE_URL` | Yes | Neon connection string with `?sslmode=require` |
| `JWT_SECRET` | Yes | Strong random string; app **refuses to start** on Render if missing or dev default |
| `CORS_ORIGINS` | Yes | Exact frontend URL(s), comma-separated, no trailing slash |
| `SENTRY_DSN` | Optional | Error monitoring; omit to disable |
| `SENTRY_TRACES_SAMPLE_RATE` | Optional | Default `0.1` when Sentry enabled |
| `RATE_LIMIT_ENABLED` | Optional | Default `true` |

Frontend (`mathassistant-frontend`): `API_BASE_URL` → backend HTTPS URL (build-time `VITE_API_BASE_URL`).

---

## 2. Staging smoke test

After deploy to staging:

- [ ] Run `scripts/smoke_test_api.ps1 -ApiBase <staging-api-url>`
- [ ] Complete [v1.0_Manual_QA.md](v1.0_Manual_QA.md) sections A–H
- [ ] Cross-browser pass (section I): Chrome + one alternate browser + mobile

---

## 3. Database

- [ ] `alembic upgrade head` succeeded in Render deploy logs
- [ ] Neon PITR / backup policy documented ([Database_Operations.md](Database_Operations.md))
- [ ] Pre-migration snapshot taken before first prod migration (if applicable)

---

## 4. CI gate (must be green on `main`)

- [ ] `pytest backend/tests/` (549+ tests)
- [ ] `alembic upgrade head` + `alembic check` (no migration drift)
- [ ] `npm test` + `npm run build` in `frontend/`

---

## 5. Security

- [ ] No `.env` or secrets committed (see [Production_Security.md](Production_Security.md))
- [ ] `JWT_SECRET` unique per environment (staging ≠ production)
- [ ] `CORS_ORIGINS` locked to your frontend host(s) only
- [ ] Rate limits active (`RATE_LIMIT_ENABLED` not `false` on Render)
- [ ] Admin `POST /problem` tested with non-admin token → `403`

---

## 6. Production promotion

- [ ] Staging checklist complete
- [ ] Merge to `main` → production Render auto-deploy
- [ ] Repeat smoke tests on production URLs
- [ ] Tag `v1.0.0` and publish GitHub release using [releases/v1.0.0_RELEASE_NOTES.md](releases/v1.0.0_RELEASE_NOTES.md)

---

## 7. Post-release

- [ ] Update demo URL in `documentation/Interview_Help.txt` if needed
- [ ] Monitor Sentry for first-hour errors (if configured)
