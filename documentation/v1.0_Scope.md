# MathAssistant v1.0 — Scope

**Product:** Algebra Co-Solving v1.0  
**Last updated:** June 22, 2026

One-page reference for what v1.0 ships versus what is explicitly deferred.

---

## In scope (v1.0)

| Area | What ships |
|------|------------|
| **Domain** | Single-variable and simple multi-step **algebra** simplification (distribution, FOIL, combine like terms). |
| **Tutoring loop** | Enter problem → start session → submit steps → correctness, error type, escalating hints → session complete or reveal after attempt limits. |
| **Validation** | Deterministic SymPy pipeline; canonical single- and multi-hop solution paths; skip-ahead when a later canonical step is submitted. |
| **Input** | Structured **keyboard algebra** (`^` exponents, parentheses, `+ - * /`). Text-like / word input rejected. |
| **Calculator UI** | Problem entry and **active session** step entry: keypad, preview, heuristic checks, **Use Expression** fills the target field. |
| **Problem library** | Seeded PostgreSQL problems; `GET /sample-problem`; optional custom expression via `POST /start-session`. |
| **Persistence** | Sessions, attempts, solution paths on PostgreSQL (Neon in prod). Alembic migrations on deploy. |
| **Frontend** | React + TypeScript + Vite; KaTeX for display; Render static site. |
| **Backend** | FastAPI on Render; `/health`, `/ready`, session APIs. |
| **Auth foundation** | `users` table, bcrypt + JWT **utilities**, migration, unit tests. **No** login/register UI or protected routes yet. |
| **Ops** | GitHub Actions CI; production + staging Render blueprints; optional Sentry. |

---

## Deferred (not v1.0)

| Area | Notes |
|------|--------|
| **OCR / image input** | Handwritten or photo problems; requires separate capture and recognition pipeline. |
| **Calculus, geometry, statistics** | Different engines, notation, and validation rules. |
| **LLM hints at runtime** | MVP hints are rule-based from error taxonomy; natural-language or adaptive LLM hints are a later layer on top of SymPy ground truth. |
| **Automatic problem generation** | No runtime LLM or procedural generator; library + user-typed expressions only. |
| **Email / activity summaries** | Opt-in weekly digests; see `Post_MVP_Features.md`. |
| **Auth product** | Register, login, JWT middleware, `user_id` on sessions, admin-gated `POST /problem`. Foundation exists; endpoints and UI are next phase. |
| **Extended problem parser** | sqrt/mod/inequalities for **new** problems — step_engine not extended; UI documents limits. |
| **Equations / inequalities** | Comparisons (`<`, `>`, `=`) and sqrt/mod rejected for new problems in v1.0. |
| **Chatbot / full-answer mode** | Product remains step-validation tutor, not an answer engine. |
