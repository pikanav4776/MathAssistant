# MathAssistant v1.0 — Manual QA Script

Step-by-step script for staging and production smoke tests before tagging `v1.0.0`.

**Time:** ~20–30 minutes full pass · **Prerequisites:** deployed frontend + backend URLs, browser devtools open (Network tab)

Replace placeholders:

| Placeholder | Example |
|-------------|---------|
| `{API}` | `https://mathassistant-api-staging.onrender.com` |
| `{APP}` | `https://mathassistant-frontend-staging.onrender.com` |

Automated API checks: `scripts/smoke_test_api.ps1 -ApiBase {API}`

---

## A. API-only (no browser)

- [ ] `GET {API}/health` → `200`, body `{"status":"ok"}`
- [ ] `GET {API}/ready` → `200`, `"db":"connected"`
- [ ] Run `scripts/smoke_test_api.ps1 -ApiBase {API}` → all steps pass

---

## B. Guest tutoring flow

1. Open `{APP}` — subtitle shows **Algebra Co-Solving (v1.0)**.
2. **Starter problems:** open panel, set topic **Multihop**, click `2(x+3)+4` (or `mhop_001`).
3. Confirm session view loads with step `1/N`.
4. Submit correct first step: `2x+6` → green feedback, advances to step 2.
5. Submit final step: `2x+10` → session completes.
6. Click **Try another** → returns to problem selection.

**Pass:** no console errors; Network requests return `200`.

---

## C. Calculator + custom problem

1. On problem selection, use calculator keypad to build `2(x+3)`.
2. Click **Use Expression** → problem field fills.
3. Click **Start Session**.
4. Submit `2x+6` → correct.
5. Refresh the page mid-session.
6. **Pass:** session restores at the same step (resume).

---

## D. Wrong-step and give-up

1. Start `2(x+3)` from starter list or typed input.
2. Submit wrong step `2x+3` → incorrect feedback with hint (not a crash).
3. Click **Reveal solution** → confirm dialog → complete view shows expected final.
4. **Pass:** friendly message; no raw SymPy errors in UI.

---

## E. Text-like input rejection

1. Type `hello world` in problem field.
2. Click **Start Session**.
3. **Pass:** error message about math notation only (no session started).

---

## F. Unsupported problem

1. Type `sqrt(x)` or `x<5` as a new problem.
2. Click **Start Session**.
3. **Pass:** friendly unsupported-problem message (not a generic 500).

---

## G. Authentication

1. **Register** a new account (unique email).
2. Confirm **Recent sessions** panel appears when signed in.
3. Start a session while signed in.
4. Refresh page — session resumes (owned session).
5. Complete or give up — session appears in history after finalize.
6. **Logout** — history panel hides.
7. **Pass:** Bearer token on API calls; owned session not accessible after logout from another browser without login.

---

## H. Random example

1. Click **Try a random example** several times.
2. **Pass:** different problems may load; each starts a valid session.
